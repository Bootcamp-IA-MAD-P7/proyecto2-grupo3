# Daily Scrum - 27/05/2026

Sprint 1 - MVP Esencial

## Resumen del equipo
El equipo avanzó en la consolidación técnica del backend y realizó una revisión del estado real del repositorio para detectar ramas, cambios y posibles conflictos antes de seguir incorporando código al proyecto.

La prioridad del día fue pasar de la planificación inicial a una base técnica más estable, comprobando qué partes del backend estaban ya integradas en `main` y qué ramas no debían fusionarse directamente.

## Qué hicimos ayer
- Fusionamos y revisamos documentación inicial del proyecto.
- Dejamos el README con una estructura más profesional.
- Documentamos el contexto de negocio del sistema de gestión para escape rooms.
- Confirmamos que Swagger estaba disponible automáticamente en `/docs`.
- Validamos el endpoint inicial `GET /health`.
- Avanzamos en la configuración inicial del backend con FastAPI.
- Iniciamos la preparación de conexión a base de datos.

## Qué hicimos hoy
- Revisamos el estado real de `main`.
- Revisamos ramas activas y Pull Requests cerradas en GitHub.
- Detectamos ramas antiguas o conflictivas que no debían fusionarse directamente.
- Confirmamos que la API podía importarse correctamente desde `backend.main`.
- Revisamos la estructura existente de `backend/core`, `backend/models` y `backend/schemas`.
- Confirmamos que los modelos y schemas principales estaban creados para `Sala`, `Cliente`, `Sesion` y `Reserva`.
- Revisamos que el logging básico y el manejo global de errores estaban integrados.
- Limpiamos archivos generados que no debían formar parte del repositorio, como logs y carpetas `__pycache__`.
- Actualizamos `.gitignore` para evitar subir archivos temporales o de entorno.
- Acordamos que las ramas de CRUD conflictivas se usarían solo como referencia y que el CRUD debía rehacerse desde una rama limpia.

## Bloqueos
- La conexión a base de datos estaba preparada a nivel de código, pero todavía no se había validado contra una base de datos real.
- La implementación de CRUD quedaba pendiente hasta decidir y configurar correctamente la base de datos.
- Algunas ramas activas contenían cambios incompatibles con la estructura actual de `main`, por lo que no se podían fusionar sin revisión.
